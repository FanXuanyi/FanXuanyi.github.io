React组件的生命周期

React

React的核心概念那么多，生命周期就是其中之一。

React生命周期（life cycle）可以分为挂载、渲染、卸载这几个阶段。
当渲染后的组件需要更新时，我们会重新去渲染组件，直至卸载。

我们可以把React生命周期分成两类：

  • 当组件在挂载或卸载时；
  • 当组件接收新的数据时，即组件更新时。

1、组件的挂载

四个生命周期方法：

  • constructor
  • componentWillMount：在render方法之前执行，代表渲染前的时刻。
  • render
  • componentDidMount：在render方法之后执行，代表渲染后的时刻。

这个过程主要做组件状态的初始化。包括读取初始state和props以及两个组件生命周期方法，这些都只会在组件初始化时运行一次。

2、组件的卸载

一个生命周期函数：

  • componentWillUnmount：在该方法中执行一些清理方法，如事件回收或清除定时器。

3、数据更新过程

1）组件自身的state更新

四个生命周期函数：

  • shouldComponentUpdate：接收需要更新的props和state，让开发者增加必要的条件判断，让其在需要使更新，不需要时不更新。当方法返回false时，组件不再执行生命周期方法。
  • componentWillUpdate：更新渲染前的的时刻。不能在此方法中执行setState。
  • render：渲染。
  • componentDidUpdate：更新渲染后的时刻。

2）组件是由父组件更新props而更新

在shouldComponentUpdate之前先执行componentWillReceiveProps方法。此方法可以作为React在props传入后，渲染之前setState的机会。在此方法中调用setState是不会二次渲染的。
